# Idea Brief Template

Use this template when the human lead wants the PM to investigate a possible feature or product change.

Keep the first section lightweight. The PM can expand the document after intake.

---

## CEO intake

### Idea / hypothesis

What might be useful?

### Why it may matter

Why investigate this now?

### Target user or workflow

Who benefits, and in what situation?

### Constraints or non-negotiables

What cannot be compromised?

### Questions for the PM

What should the PM answer before this moves forward?

---

## PM discovery

### Problem framing

What user problem is this actually solving?

### UX / behavior notes

What would the user see or feel? What defaults or edge cases matter?

### Recommendation

- Pursue
- Defer
- Reject

Explain why.

### Risks / open questions

What still needs to be understood?

### Suggested next artifact

If this should proceed, what should come next?

- user-facing doc update
- product/spec doc
- architect technical spec
- test plan
- other
